# Feature 4.3: In-App Notifications UI

**Status:** Pending
**Effort:** Medium
**Phase:** 4 - Notifications

## Overview

Display in-app notifications to users through a notification bell in the header with unread count badge, a dropdown showing recent notifications, and a full notifications index page for viewing all notifications.

## Requirements

### Functional Requirements
- Notification bell icon in header
- Unread notification count badge
- Dropdown showing recent notifications
- Full notifications index page
- Mark as read functionality (individual and bulk)
- Optional: Real-time updates using Pusher/Websockets

### UI Components
- Header notification bell with badge
- Dropdown with recent notifications (5-10 items)
- "View All" link to full notifications page
- Full index page with pagination
- Mark as read/unread actions
- Delete notification action

## Notification Display Format

Each notification in the list should show:
- Icon based on notification type
- Title/summary of the notification
- Relative timestamp (e.g., "2 hours ago")
- Read/unread status indicator
- Link to related resource (shift, leave request, etc.)

## API Endpoints

```
GET  /notifications           - List all notifications (paginated)
GET  /notifications/unread    - Get unread count
POST /notifications/{id}/read - Mark single as read
POST /notifications/read-all  - Mark all as read
DELETE /notifications/{id}    - Delete notification
```

## Real-Time Updates (Optional)

If implementing real-time updates:
- Use Laravel Echo with Pusher or Websockets
- Broadcast new notifications to user's private channel
- Update badge count without page refresh
- Show toast notification for new items

## Dependencies

- Laravel's built-in notification system (database driver)
- User model with notifications relationship
- App layout component
- Feature 4.1 (Additional Notifications)
