# Feature 4.2: Notification Preferences UI

**Status:** Pending
**Effort:** Small
**Phase:** 4 - Notifications

## Overview

Allow users to manage their notification preferences through a settings interface. Users can toggle different notification types and channels (email, push, in-app) according to their preferences.

## Requirements

### Functional Requirements
- Toggle email notifications per notification type
- Toggle push notifications per notification type (for future mobile)
- Toggle in-app notifications per notification type
- Preferences are saved per user
- System respects preferences when sending notifications

### Notification Types to Configure
- Shift assignments
- Shift changes
- Leave request status
- Swap requests received
- Swap request responses
- Shift reminders
- Schedule published

    'swap_response' => ['email' => true, 'database' => true, 'push' => false],

## UI Layout

The preferences page should display:
- Table with notification types as rows
- Columns for each channel (Email, In-App, Push)
- Toggle switches for each combination
- Save button to persist changes

## Dependencies

- User model
- NotificationPreference model or user_settings column
- All notification classes from Feature 4.1
