# UAT: Feature 9.6 - Pull-to-Refresh & Swipe Actions

## Test Environment Setup

**Prerequisites:**
- Admin and employee user accounts
- Mobile device with touch screen
- Manager account for approval testing
- Pending approval items (leave requests, swaps)

**Test URL:** `/dashboard`, `/schedule`, `/approvals`

---

## Test Cases

### TC-9.6.1: Pull-to-Refresh on Dashboard

**Steps:**
1. Log in as an employee on mobile
2. Navigate to Dashboard
3. Pull down from the top of the content area
4. Release when refresh indicator appears

**Expected Results:**
- [ ] Pull gesture is detected
- [ ] Loading indicator appears during pull
- [ ] Data refreshes when released
- [ ] New data appears (if any changes)
- [ ] Indicator disappears after refresh

---

### TC-9.6.2: Pull-to-Refresh on Schedule

**Steps:**
1. Navigate to Schedule on mobile
2. Perform pull-to-refresh gesture
3. Release

**Expected Results:**
- [ ] Schedule data refreshes
- [ ] Any new shifts appear
- [ ] Smooth animation during refresh
- [ ] Page state preserved (current week view)

---

### TC-9.6.3: Visual Feedback During Pull

**Steps:**
1. Start pull-to-refresh gesture
2. Hold before releasing
3. Observe visual elements

**Expected Results:**
- [ ] Spinner or loading icon appears
- [ ] Pull distance indicator (optional)
- [ ] Color change or animation feedback
- [ ] Clear indication of "release to refresh" state

---

### TC-9.6.4: Swipe-to-Approve (Manager)

**Steps:**
1. Log in as a manager
2. Navigate to pending approvals (leave requests)
3. Swipe right on a pending request

**Expected Results:**
- [ ] Swipe gesture is detected
- [ ] Green "Approve" action revealed
- [ ] Completing swipe triggers approval
- [ ] Request is approved

---

### TC-9.6.5: Swipe-to-Deny (Manager)

**Steps:**
1. Log in as a manager
2. Navigate to pending approvals
3. Swipe left on a pending request

**Expected Results:**
- [ ] Swipe gesture is detected
- [ ] Red "Deny" action revealed
- [ ] Completing swipe triggers denial
- [ ] Request is denied

---

### TC-9.6.6: Swipe to View Shift Details

**Steps:**
1. Log in as an employee
2. Navigate to Schedule
3. Swipe on a shift card

**Expected Results:**
- [ ] Swipe reveals action options OR
- [ ] Swipe opens shift details drawer
- [ ] Smooth animation during swipe
- [ ] Can swipe back to close

---

### TC-9.6.7: Swipe Threshold Behavior

**Steps:**
1. Start swipe gesture
2. Swipe only partially (not full threshold)
3. Release

**Expected Results:**
- [ ] Action is NOT triggered
- [ ] Card snaps back to original position
- [ ] No partial action taken
- [ ] Smooth snap-back animation

---

### TC-9.6.8: Multiple Swipe Actions

**Steps:**
1. Swipe to reveal actions on an item
2. Swipe on a different item
3. Observe first item

**Expected Results:**
- [ ] First item's actions close
- [ ] Second item's actions open
- [ ] Only one item shows actions at a time
- [ ] No overlapping swipe states

---

### TC-9.6.9: Touch Gesture on Non-Swipable Items

**Steps:**
1. Navigate to a page with non-swipable content
2. Attempt swipe gestures

**Expected Results:**
- [ ] No swipe action triggered
- [ ] Normal scrolling works
- [ ] No confusing UI behavior

---

### TC-9.6.10: Pull-to-Refresh During Loading

**Steps:**
1. Start pull-to-refresh
2. While loading, try to pull again

**Expected Results:**
- [ ] Second pull is ignored
- [ ] Loading continues normally
- [ ] No duplicate refresh requests
- [ ] UI remains stable

---

### TC-9.6.11: Swipe Action Undo/Cancel

**Steps:**
1. Start swipe to reveal action
2. Before completing action, swipe back
3. Observe result

**Expected Results:**
- [ ] Action is cancelled
- [ ] Item returns to normal state
- [ ] No action was taken
- [ ] Can try again

---

### TC-9.6.12: Touch Gesture Accessibility

**Steps:**
1. Enable accessibility features on device
2. Attempt to use swipe actions

**Expected Results:**
- [ ] Alternative action method available
- [ ] Screen reader announces swipe options
- [ ] Actions accessible without gestures
