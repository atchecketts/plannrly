# UAT: Feature 9.10 - Accessibility Improvements

## Test Environment Setup

**Prerequisites:**
- Screen reader software (VoiceOver on iOS/Mac, TalkBack on Android, NVDA on Windows)
- Keyboard-only navigation ability
- Automated accessibility testing tool (axe, WAVE, Lighthouse)
- Color contrast analyzer tool

**Test URL:** `/` (all pages)

---

## Test Cases

### TC-9.10.1: ARIA Labels on Interactive Elements

**Steps:**
1. Inspect buttons, links, and form controls
2. Check for aria-label or aria-labelledby attributes
3. Verify labels are descriptive

**Expected Results:**
- [ ] All buttons have accessible names
- [ ] Icon-only buttons have aria-label
- [ ] Form inputs have associated labels
- [ ] Images have alt text

---

### TC-9.10.2: Keyboard Navigation

**Steps:**
1. Starting from page load, use only keyboard
2. Tab through all interactive elements
3. Verify every element is reachable

**Expected Results:**
- [ ] Can tab to all interactive elements
- [ ] Tab order is logical (top-to-bottom, left-to-right)
- [ ] Can activate buttons with Enter/Space
- [ ] Can navigate menus with arrow keys

---

### TC-9.10.3: Skip Navigation Links

**Steps:**
1. Load page and press Tab immediately
2. Look for skip link

**Expected Results:**
- [ ] "Skip to main content" link appears first
- [ ] Activating link skips to main content
- [ ] Focus moves past navigation
- [ ] Visible when focused

---

### TC-9.10.4: Screen Reader Testing (VoiceOver)

**Steps:**
1. Enable VoiceOver on iOS/Mac
2. Navigate through the application
3. Listen to announcements

**Expected Results:**
- [ ] All content is announced
- [ ] Interactive elements are identified
- [ ] Form inputs announce label and state
- [ ] Dynamic content changes announced

---

### TC-9.10.5: Screen Reader Testing (TalkBack)

**Steps:**
1. Enable TalkBack on Android device
2. Navigate through the application
3. Verify all content is accessible

**Expected Results:**
- [ ] Swipe navigation works through content
- [ ] All elements are reachable
- [ ] Actions are clearly announced
- [ ] No trapped focus areas

---

### TC-9.10.6: Color Contrast Compliance

**Steps:**
1. Use color contrast analyzer tool
2. Check text against backgrounds
3. Check for various UI elements

**Expected Results:**
- [ ] Normal text: minimum 4.5:1 ratio
- [ ] Large text: minimum 3:1 ratio
- [ ] UI components: minimum 3:1 ratio
- [ ] Meets WCAG AA standards

---

### TC-9.10.7: Focus Indicators

**Steps:**
1. Tab through interactive elements
2. Observe focus styling

**Expected Results:**
- [ ] Focus indicator is visible
- [ ] Sufficient contrast for focus ring
- [ ] Focus indicator on all interactive elements
- [ ] Custom focus styles match brand (if used)

---

### TC-9.10.8: Reduced Motion Preference

**Steps:**
1. Enable "Reduce motion" in OS accessibility settings
2. Load application
3. Observe animations

**Expected Results:**
- [ ] Animations are reduced or removed
- [ ] Transitions are simplified
- [ ] Content still functions correctly
- [ ] No essential information lost

---

### TC-9.10.9: Heading Hierarchy

**Steps:**
1. Use accessibility tool to view heading structure
2. Verify heading levels are sequential

**Expected Results:**
- [ ] One H1 per page
- [ ] Headings follow logical order (H1, H2, H3...)
- [ ] No skipped heading levels
- [ ] Headings describe content sections

---

### TC-9.10.10: Form Accessibility

**Steps:**
1. Navigate to forms (login, leave request, etc.)
2. Test with screen reader
3. Test error handling

**Expected Results:**
- [ ] Labels associated with inputs
- [ ] Required fields indicated
- [ ] Error messages announced
- [ ] Error messages describe how to fix

---

### TC-9.10.11: Modal/Dialog Accessibility

**Steps:**
1. Open a modal dialog
2. Test keyboard navigation within modal
3. Test screen reader behavior

**Expected Results:**
- [ ] Focus trapped within modal
- [ ] Escape key closes modal
- [ ] Focus returns to trigger element on close
- [ ] Modal has accessible role and title

---

### TC-9.10.12: Table Accessibility

**Steps:**
1. Navigate to pages with data tables
2. Test with screen reader
3. Verify table structure

**Expected Results:**
- [ ] Tables have proper headers (th)
- [ ] Headers associated with cells (scope)
- [ ] Caption provided if helpful
- [ ] Screen reader announces row/column

---

### TC-9.10.13: Automated Accessibility Audit

**Steps:**
1. Run axe or Lighthouse accessibility audit
2. Review all findings
3. Document issues

**Expected Results:**
- [ ] No critical accessibility violations
- [ ] No serious violations
- [ ] Minor issues documented for improvement
- [ ] Accessibility score > 90

---

### TC-9.10.14: Text Resize

**Steps:**
1. Zoom browser to 200%
2. Navigate through application
3. Check for content issues

**Expected Results:**
- [ ] Text remains readable at 200%
- [ ] No text truncation
- [ ] No overlapping content
- [ ] Layout adapts appropriately

---

### TC-9.10.15: Alternative Input Methods

**Steps:**
1. Test with voice control (if available)
2. Test with switch control (if available)

**Expected Results:**
- [ ] Actions can be performed with voice
- [ ] Interactive elements are clickable via voice
- [ ] Alternative navigation methods work
