# UAT: Feature 28.4 - In-App Help & AI Chat

## Test Environment Setup

**Prerequisites:**
- Business admin user on Professional+ tier
- Business admin user on Starter tier
- Regular employee user
- AI service integration configured (OpenAI/Claude API)
- Help content populated in knowledge base

**Test URL:** Help icon accessible from any page

---

## Test Cases

### TC-28.4.1: View Contextual Help Tooltip

**Steps:**
1. Log in as any user
2. Navigate to a complex feature page
3. Find help icon (?) next to a field or section
4. Hover over or click the help icon

**Expected Results:**
- [ ] Help icon is visible
- [ ] Tooltip appears on interaction
- [ ] Tooltip explains the feature/field
- [ ] Tooltip is contextually relevant
- [ ] Can dismiss tooltip easily

---

### TC-28.4.2: Help Tooltips Across Features

**Steps:**
1. Navigate to Leave Request form
2. Find help tooltips
3. Navigate to Shift creation
4. Find help tooltips
5. Navigate to Settings

**Expected Results:**
- [ ] Help available on Leave Request page
- [ ] Help available on Shift creation
- [ ] Help available on Settings pages
- [ ] Consistent help icon placement
- [ ] Content is relevant to each page

---

### TC-28.4.3: Open Help Center/Knowledge Base

**Steps:**
1. Log in as any user
2. Find "Help" link in menu or footer
3. Click to open Help Center

**Expected Results:**
- [ ] Help Center opens (modal or new page)
- [ ] Shows categories of help topics
- [ ] Search functionality available
- [ ] Articles are readable
- [ ] Navigation is intuitive

---

### TC-28.4.4: Search Knowledge Base

**Steps:**
1. Open Help Center
2. Enter search term: "leave request"
3. View search results
4. Click on a result

**Expected Results:**
- [ ] Search input is available
- [ ] Results appear as you type or on submit
- [ ] Results are relevant to search term
- [ ] Clicking result shows full article
- [ ] Search handles no results gracefully
