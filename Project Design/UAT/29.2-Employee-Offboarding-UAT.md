# UAT: Feature 29.2 - Employee Offboarding

## Test Environment Setup

**Prerequisites:**
- Admin user with employee management permissions
- Employee user to be offboarded
- Employee with future scheduled shifts
- Employee with pending tasks/approvals
- Employee with messages and data history

**Test URL:** `/employees/{id}/offboard`

---

## Test Cases

### TC-29.2.1: Initiate Offboarding

**Steps:**
1. Log in as admin
2. Navigate to employee management
3. Select an employee
4. Find "Offboard" or "Terminate" option
5. Click to initiate offboarding

**Expected Results:**
- [ ] Offboarding option is available
- [ ] Opens offboarding wizard/form
- [ ] Shows employee details
- [ ] Requires confirmation

---

### TC-29.2.2: Select Offboarding Reason

**Steps:**
1. Initiate offboarding
2. View available reasons
3. Select "Resignation"
4. Try selecting "Termination"
5. Try selecting "End of Contract"

**Expected Results:**
- [ ] Multiple reason options available
- [ ] Can add notes/comments
- [ ] Last working day field available
- [ ] Reason is required

---

### TC-29.2.3: Set Last Working Day

**Steps:**
1. During offboarding
2. Set last working day to a future date
3. Set last working day to today
4. Try setting to a past date

**Expected Results:**
- [ ] Date picker available
- [ ] Future dates accepted
- [ ] Today accepted
- [ ] Past dates may show warning
- [ ] Date affects shift removal

---

### TC-29.2.4: Review Future Shifts

**Steps:**
1. Offboard employee with future shifts
2. Review the impact summary
3. See list of shifts to be removed

**Expected Results:**
- [ ] Shows count of future shifts
- [ ] Lists upcoming shifts
- [ ] Option to reassign vs. delete
- [ ] Coverage needed indicator

---

### TC-29.2.5: Automatic Shift Removal

**Steps:**
1. Complete offboarding for employee with shifts
2. Check schedule after offboarding

**Expected Results:**
- [ ] Future shifts are unassigned
- [ ] Shifts become open/unassigned
- [ ] Past shifts are preserved
- [ ] Schedule updates correctly

---

### TC-29.2.6: Pending Task Reassignment

**Steps:**
1. Offboard employee with pending approvals
2. Review pending items
3. Assign pending items to another user

**Expected Results:**
- [ ] Pending approvals listed
- [ ] Can reassign to another manager
- [ ] Items transfer correctly
- [ ] No orphaned tasks

---

### TC-29.2.7: Message Archival

**Steps:**
1. Offboard employee with message history
2. Complete offboarding
3. Check if messages are archived

**Expected Results:**
- [ ] Messages are preserved/archived
- [ ] Not deleted immediately
- [ ] Admin can access if needed
- [ ] Complies with data retention

---

### TC-29.2.8: Access Revocation

**Steps:**
1. Complete offboarding
2. Try logging in as offboarded employee
3. Check system access

**Expected Results:**
- [ ] Login is disabled
- [ ] Clear message shown (account deactivated)
- [ ] No access to any features
- [ ] Session terminated if active

---

### TC-29.2.9: Soft Delete Implementation

**Steps:**
1. Complete offboarding
2. Check employee in admin view
3. Verify data is preserved

**Expected Results:**
- [ ] Employee record not hard deleted
- [ ] Historical data preserved
- [ ] Reports still include past data
- [ ] Can distinguish active vs. offboarded

---

### TC-29.2.10: Offboarding Task Checklist

**Steps:**

**Expected Results:**

### TC-29.2.11: Offboarding Notifications

**Expected Results:**

### TC-29.2.12: Cancel Offboarding (Before Completion)
