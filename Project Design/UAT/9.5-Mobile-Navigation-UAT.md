# UAT: Feature 9.5 - Mobile Bottom Navigation

## Test Environment Setup

**Prerequisites:**
- Admin and employee user accounts
- Mobile device or browser in mobile viewport
- Various screen sizes for testing

**Test URL:** `/dashboard` (or any authenticated page)

---

## Test Cases

### TC-9.5.1: Bottom Navigation Visibility on Mobile

**Steps:**
1. Log in as an employee
2. View page on mobile device (or resize browser to mobile width)

**Expected Results:**
- [ ] Bottom navigation bar is visible
- [ ] Navigation is fixed to bottom of screen
- [ ] Contains 5 items: Home, Schedule, Clock, Requests, Profile
- [ ] Clock button is prominently centered

---

### TC-9.5.2: Bottom Navigation Hidden on Desktop

**Steps:**
1. Log in as an employee
2. View page on desktop (wide viewport)

**Expected Results:**
- [ ] Bottom navigation is NOT visible
- [ ] Sidebar navigation is used instead
- [ ] No duplicate navigation elements

---

### TC-9.5.3: Navigation Item Active States

**Steps:**
1. On mobile, navigate to each section:
   - Home (Dashboard)
   - Schedule
   - Requests
   - Profile
2. Observe active state

**Expected Results:**
- [ ] Active item is visually highlighted
- [ ] Only one item active at a time
- [ ] Active state updates on navigation
- [ ] Color/icon change indicates active state

---

### TC-9.5.4: Clock Button Functionality

**Steps:**
1. On mobile, tap the center Clock button
2. Observe the action

**Expected Results:**
- [ ] Clock widget/modal opens
- [ ] Shows current clock status
- [ ] Can clock in/out directly
- [ ] Prominent tap target (larger than other buttons)

---

### TC-9.5.5: Touch Target Size Compliance

**Steps:**
1. On mobile, examine each navigation item
2. Use browser dev tools to measure tap target sizes

**Expected Results:**
- [ ] Each touch target is minimum 44x44 pixels
- [ ] Adequate spacing between targets
- [ ] No accidental mis-taps during normal use

---

### TC-9.5.6: Safe Area Insets (Notched Devices)

**Steps:**
1. View on device with notch/home indicator (iPhone X+, etc.)
2. Observe bottom navigation positioning

**Expected Results:**
- [ ] Navigation respects safe area insets
- [ ] Content not obscured by home indicator
- [ ] Navigation sits above device UI elements

---

### TC-9.5.7: Mobile Clock Widget

**Steps:**
1. Tap Clock button to open clock widget
2. Observe the widget display

**Expected Results:**
- [ ] Shows current time prominently
- [ ] Displays clock status (Clocked In/Out)
- [ ] Large Clock In/Out button
- [ ] Shows current shift details if applicable

---

### TC-9.5.8: Clock Widget Actions

**Steps:**
1. Open clock widget while clocked out
2. Tap Clock In button
3. Open widget while clocked in
4. Tap Clock Out button

**Expected Results:**
- [ ] Clock In action works from widget
- [ ] Clock Out action works from widget
- [ ] Status updates immediately
- [ ] Success confirmation shown

---

### TC-9.5.9: Mobile Shift Card Display

**Steps:**
1. Navigate to Schedule on mobile
2. View shift list items

**Expected Results:**
- [ ] Shift cards display cleanly
- [ ] Date and time clearly visible
- [ ] Location shown
- [ ] Role/position shown
- [ ] Cards are tappable for details

---

### TC-9.5.10: Haptic Feedback on Clock Actions

**Steps:**
1. On supported device, perform clock in/out
2. Note physical feedback

**Expected Results:**
- [ ] Haptic vibration on clock action (if device supports)
- [ ] Feedback confirms button press
- [ ] No feedback on devices without support

---

### TC-9.5.11: Navigation Scroll Behavior

**Steps:**
1. On mobile, scroll through a long page
2. Observe bottom navigation

**Expected Results:**
- [ ] Navigation stays fixed at bottom
- [ ] Does not scroll with content
- [ ] Always accessible during scroll
- [ ] Content scrolls behind navigation

---

### TC-9.5.12: Landscape Orientation

**Steps:**
1. Rotate device to landscape mode
2. Observe bottom navigation

**Expected Results:**
- [ ] Navigation adapts to landscape
- [ ] Still accessible and usable
- [ ] No layout breaking
- [ ] Icons and labels remain visible

---
