# UAT: Feature 20.1 - Onboarding Wizard

## Test Environment Setup

**Prerequisites:**
- Fresh tenant account (newly registered)
- Access to tenant admin credentials
- Test email addresses for team invitations

**Test URL:** `/onboarding`

---

## Test Cases

### TC-20.1.1: Onboarding Wizard Trigger on New Tenant

**Steps:**
1. Register a new tenant account
2. Complete email verification
3. Log in for the first time

**Expected Results:**
- [ ] Onboarding wizard launches automatically
- [ ] Cannot skip or dismiss wizard without completing key steps
- [ ] Progress indicator shows current step
- [ ] Welcome message is displayed

---

### TC-20.1.2: Step 1 - Organization Details

**Steps:**
1. Start the onboarding wizard
2. Enter organization name
3. Enter business type/industry
4. Upload organization logo (optional)
5. Click "Next"

**Expected Results:**
- [ ] Required fields are validated
- [ ] Organization name is saved
- [ ] Logo preview is shown if uploaded
- [ ] Progress to Step 2 on success

---

### TC-20.1.3: Step 2 - First Location

**Steps:**
1. Complete Step 1
2. Enter location name
3. Enter location address
4. Set timezone
5. Click "Next"

**Expected Results:**
- [ ] Location is created successfully
- [ ] Address validation (if applicable)
- [ ] Timezone dropdown works correctly
- [ ] Progress to Step 3 on success

---

### TC-20.1.4: Step 3 - Departments (Skippable)

**Steps:**
1. Complete Step 2
2. View the departments step
3. Option A: Add one or more departments
4. Option B: Click "Skip for now"

**Expected Results:**
- [ ] Skip option is clearly visible
- [ ] Can add multiple departments
- [ ] Departments are created if added
- [ ] Progress continues whether skipped or completed

---

### TC-20.1.5: Step 4 - Business Roles

**Steps:**
1. Complete Step 3 (or skip)
2. Add at least one business role
3. Set hourly rate (optional)
4. Click "Next"

**Expected Results:**
- [ ] At least one role is required
- [ ] Multiple roles can be added
- [ ] Roles are created successfully
- [ ] Progress to Step 5 on success

---

### TC-20.1.6: Step 5 - Invite Team Members

**Steps:**
1. Complete Step 4
2. Enter email addresses for team members
3. Assign roles to invitees
4. Click "Send Invitations" or "Skip"

**Expected Results:**
- [ ] Can add multiple email addresses
- [ ] Role assignment works for each invitee
- [ ] Invitations are sent successfully
- [ ] Skip option available if no invites needed
- [ ] Progress to Step 6 on success

---

### TC-20.1.7: Step 6 - First Schedule (Optional)

**Steps:**
1. Complete Step 5
2. Option to create first schedule or skip
3. If creating: select date range and location
4. Complete or skip

**Expected Results:**
- [ ] Clear option to skip or create
- [ ] Schedule created if completed
- [ ] Basic schedule interface is shown
- [ ] Wizard completes on this step

---

### TC-20.1.8: Progress Indicator Accuracy

**Steps:**
1. Navigate through all wizard steps
2. Observe progress indicator at each step

**Expected Results:**
- [ ] Progress bar shows correct completion percentage
- [ ] Step numbers are clearly displayed
- [ ] Current step is highlighted
- [ ] Completed steps show checkmarks

---

### TC-20.1.9: Wizard Resume After Browser Close

**Steps:**
1. Start onboarding wizard
2. Complete Steps 1-3
3. Close the browser/tab
4. Log back in

**Expected Results:**
- [ ] Wizard resumes at the correct step
- [ ] Previously entered data is preserved
- [ ] No need to re-enter completed steps
- [ ] Progress indicator reflects saved progress

---

### TC-20.1.10: Sample Data Loader

**Steps:**
1. During onboarding, look for "Load Sample Data" option
2. Click to load sample data
3. Review what sample data is created

**Expected Results:**
- [ ] Sample employees are created
- [ ] Sample shifts/schedules are created
- [ ] Sample data is clearly labeled as sample
- [ ] User understands this is demo data

---

### TC-20.1.11: Clear Sample Data Option

**Steps:**
1. After loading sample data, find "Clear Sample Data" option
2. Click to clear sample data
3. Confirm the action

**Expected Results:**
- [ ] Confirmation dialog appears
- [ ] All sample data is removed
- [ ] Real data (if any) is preserved
- [ ] Success message is shown

---

### TC-20.1.12: Persistent Checklist After Wizard

**Steps:**
1. Complete the onboarding wizard
2. Navigate to the main dashboard
3. Look for onboarding checklist

**Expected Results:**
- [ ] Checklist shows remaining setup tasks
- [ ] Completed tasks are marked
- [ ] Checklist is dismissible after completion
- [ ] Quick links to incomplete tasks

---

### TC-20.1.13: Contextual Help and Videos

**Steps:**
1. During wizard, look for help icons/links
2. Click on help for each step
3. View any embedded videos

**Expected Results:**
- [ ] Help content is relevant to current step
- [ ] Videos play correctly (if present)
- [ ] Help modal/tooltip doesn't block progress
- [ ] Can dismiss help and continue

---

### TC-20.1.14: Back Navigation in Wizard

**Steps:**
1. Progress to Step 4 of the wizard
2. Click "Back" to return to Step 3
3. Navigate back and forth

**Expected Results:**
- [ ] Back button works on all steps (except Step 1)
- [ ] Data entered is preserved when going back
- [ ] Can edit previous step data
- [ ] Forward progress continues normally

---

### TC-20.1.15: Validation Errors Display

**Steps:**
1. Attempt to proceed without filling required fields
2. Enter invalid data (e.g., invalid email format)
3. Observe error messages

**Expected Results:**
- [ ] Clear error messages are shown
- [ ] Error fields are highlighted
- [ ] Cannot proceed until errors are fixed
- [ ] Errors clear when corrected
