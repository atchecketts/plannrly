# UAT: Feature 16.1 - Shift Preferences

## Test Environment Setup

**Prerequisites:**
- Employee user accounts
- Admin user account
- Scheduling system active
- Coworkers in the same department

**Test URL:** `/settings/shift-preferences`

---

## Test Cases

### TC-16.1.1: Access Shift Preferences Page

**Steps:**
1. Log in as an employee
2. Navigate to account settings
3. Click on "Shift Preferences"

**Expected Results:**
- [ ] Shift preferences page loads successfully
- [ ] Current preferences displayed (or empty state)
- [ ] Options for day/time preferences visible
- [ ] Save button available

---

### TC-16.1.2: Set Day Preferences

**Steps:**
1. Navigate to shift preferences
2. Select preference levels for each day of the week
3. Save preferences

**Expected Results:**
- [ ] All days of week selectable
- [ ] Preference levels available (Strong, Mild, Neutral, Avoid)
- [ ] Preferences save successfully
- [ ] Confirmation message shown

---

### TC-16.1.3: Set Time Preferences

**Steps:**
1. Navigate to shift preferences
2. Set preferred time ranges (e.g., Morning, Afternoon, Evening)
3. Save preferences

**Expected Results:**
- [ ] Time preference options available
- [ ] Can indicate preferred shift times
- [ ] Preferences save successfully
- [ ] Multiple time preferences can be set

---

### TC-16.1.4: Preference Levels (Strong/Mild/Avoid)

**Steps:**
1. Navigate to shift preferences
2. Set different preference levels:
   - Strong preference for Monday
   - Mild preference for Tuesday
   - Avoid for Saturday
3. Save and verify

**Expected Results:**
- [ ] Strong preference clearly indicated
- [ ] Mild preference distinguished from strong
- [ ] Avoid preference shown as negative
- [ ] All levels save correctly

---

### TC-16.1.5: Add Coworker Preferences

**Steps:**
1. Navigate to shift preferences
2. Find coworker preference section
3. Add a preferred coworker
4. Add a coworker to avoid (if feature supports)
5. Save preferences

**Expected Results:**
- [ ] Coworker list shows team members
- [ ] Can mark preferred coworkers
- [ ] Can mark coworkers to avoid (optional)
- [ ] Preferences save correctly

---

### TC-16.1.6: Update Existing Preferences

**Steps:**
1. Set some shift preferences
2. Return to preferences page
3. Modify the preferences
4. Save changes

**Expected Results:**
- [ ] Existing preferences are pre-populated
- [ ] Changes can be made
- [ ] Updates save successfully
- [ ] Old preferences are replaced

---

### TC-16.1.7: Clear All Preferences

**Steps:**
1. Navigate to shift preferences with existing preferences
2. Clear all preferences (reset to neutral)
3. Save

**Expected Results:**
- [ ] Clear/reset option available
- [ ] All preferences return to neutral
- [ ] Saves without error
- [ ] Scheduling treats as no preference

---

### TC-16.1.8: Preferences Visible in Scheduling View (Admin)

**Steps:**
1. As employee, set some shift preferences
2. Log in as admin
3. Navigate to scheduling
4. View the employee's assignments

**Expected Results:**
- [ ] Admin can see employee preferences
- [ ] Preferences shown as hints/indicators
- [ ] Strong preferences highlighted
- [ ] Avoid preferences warned

---

### TC-16.1.9: Preference Satisfaction Score

**Steps:**
1. Set preferences as employee
2. After shifts are scheduled
3. View preference satisfaction score (if available)

**Expected Results:**
- [ ] Satisfaction score calculated
- [ ] Shows how well schedule matches preferences
- [ ] Score breakdown available
- [ ] Historical scores trackable

---

### TC-16.1.10: Preferences Do Not Override Availability

**Steps:**
1. Set availability as unavailable Monday
2. Set strong preference for Monday
3. Verify scheduling behavior

**Expected Results:**
- [ ] Availability takes precedence
- [ ] Cannot be scheduled on unavailable days
- [ ] Preferences only considered for available times
- [ ] No conflicts created

---

### TC-16.1.11: Tenant Isolation of Preferences

**Steps:**
1. Set preferences as employee in Tenant A
2. Log in as employee in Tenant B
3. Check preferences

**Expected Results:**
- [ ] Preferences are tenant-specific
- [ ] Cannot see other tenant's employees as coworkers
- [ ] Data isolation maintained

---

### TC-16.1.12: Validation of Preferences

**Steps:**
1. Try to save invalid preferences (if possible)
2. Submit the form

**Expected Results:**
- [ ] Invalid data is rejected
- [ ] Error messages are clear
- [ ] Valid data is preserved
- [ ] Form does not submit with errors
