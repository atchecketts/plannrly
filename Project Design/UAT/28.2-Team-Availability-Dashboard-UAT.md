# UAT: Feature 28.2 - Team Availability Dashboard

## Test Environment Setup

**Prerequisites:**
- Admin/Manager user with team visibility
- Multiple employees in the same department
- Employees with various states: scheduled, on leave, available
- Shifts assigned across the week
- Leave requests approved
- Employee skill/certification data configured

**Test URL:** `/team/availability`

---

## Test Cases

### TC-28.2.1: Access Team Availability Dashboard

**Steps:**
1. Log in as admin or manager
2. Navigate to "Team Availability" or similar menu option

**Expected Results:**
- [ ] Dashboard loads successfully
- [ ] Calendar/timeline view is visible
- [ ] Employee list is displayed
- [ ] Current week is shown by default

---

### TC-28.2.2: View Team Calendar

**Steps:**
1. Open team availability dashboard
2. View the calendar display

**Expected Results:**
- [ ] Calendar shows days across the top
- [ ] Employees listed down the side
- [ ] Each cell shows availability status
- [ ] Color coding indicates different states
- [ ] Legend explains color meanings

---

### TC-28.2.3: Understand Availability Colors

**Steps:**
1. View the dashboard
2. Identify the color legend
3. Compare with employee actual status

**Expected Results:**
- [ ] Working/Scheduled: shows assigned shift times
- [ ] Available: no shift, not on leave
- [ ] On Leave: clearly marked with leave type
- [ ] Unavailable: availability preferences say no
- [ ] Part-day shows mixed coloring

---

### TC-28.2.4: View Employee with Shift

**Steps:**
1. Find an employee with a scheduled shift
2. View their row on the calendar

**Expected Results:**
- [ ] Shift time range is visible
- [ ] Shows shift start and end time
- [ ] Shows location or role
- [ ] Cell background indicates working

---

### TC-28.2.5: View Employee on Leave

**Steps:**
1. Find an employee with approved leave
2. View their row on the calendar

**Expected Results:**
- [ ] Leave period is clearly marked
- [ ] Shows leave type (annual, sick, etc.)
- [ ] Different color from scheduled shifts
- [ ] Full-day or half-day indicated

---

### TC-28.2.6: View Employee Availability Preferences

**Steps:**
1. Find an employee who has set availability preferences
2. View their availability on the calendar
3. Check a day they marked as unavailable

**Expected Results:**
- [ ] Preference-based unavailability shown
- [ ] Different from scheduled unavailability
- [ ] Indicates preference, not confirmed
- [ ] Can still schedule if needed (with warning)

---

### TC-28.2.7: Filter by Department

**Steps:**
1. Open team availability dashboard
2. Use department filter
3. Select "Sales" department
4. Clear filter and select "Operations"

**Expected Results:**
- [ ] Department filter is available
- [ ] Filter updates employee list
- [ ] Only selected department shown
- [ ] Can clear filter to see all

---

### TC-28.2.8: Filter by Skill/Certification

**Steps:**
1. Open team availability dashboard
2. Find skill/certification filter
3. Select a skill (e.g., "Forklift Certified")
4. View filtered results

**Expected Results:**
- [ ] Skill filter is available
- [ ] Only employees with skill are shown
- [ ] Useful for specialized shift coverage
- [ ] Can combine with department filter

---

### TC-28.2.9: Gap Analysis - Understaffed Periods

**Steps:**
1. View the availability dashboard
2. Find a day/time with few available employees
3. Check for gap warnings

**Expected Results:**
- [ ] Understaffed periods are highlighted
- [ ] Shows required vs. available staff
- [ ] Warning color or icon for gaps
- [ ] Can click for more details

---

### TC-28.2.10: View Unified Day Details

**Steps:**
1. Click on a specific day column
2. View the day detail panel

**Expected Results:**

- [ ] Grouped by: Working, Available, Leave
- [ ] Shows total hours scheduled
- [ ] Shows gaps if any
- [ ] Links to create shifts

### TC-28.2.11: Quick-Assign from Availability View

**Steps:**

3. Click "Assign" or drag to create shift

**Expected Results:**
- [ ] Can initiate shift creation from dashboard

- [ ] Opens shift creation form
