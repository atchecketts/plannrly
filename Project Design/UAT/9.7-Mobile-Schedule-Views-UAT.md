# UAT: Feature 9.7 - Mobile Schedule Views

## Test Environment Setup

**Prerequisites:**
- Employee and admin user accounts
- Mobile device or mobile viewport browser
- Published shifts spanning multiple weeks
- Various shift types and locations

**Test URL:** `/schedule`

---

## Test Cases

### TC-9.7.1: Mobile Week View Layout

**Steps:**
1. Log in as an employee on mobile
2. Navigate to Schedule
3. Observe default week view

**Expected Results:**
- [ ] Week displays in vertical list format
- [ ] Each day shows as a section header
- [ ] Shifts listed under respective days
- [ ] Day without shifts shows "No shifts"

---

### TC-9.7.2: Swipe Navigation Between Weeks

**Steps:**
1. On mobile schedule view
2. Swipe left to go to next week
3. Swipe right to go to previous week

**Expected Results:**
- [ ] Swipe gesture navigates weeks
- [ ] Smooth transition animation
- [ ] Week header updates to new week
- [ ] Correct shifts display for navigated week

---

### TC-9.7.3: Day View on Mobile

**Steps:**
1. On mobile schedule
2. Tap on a specific day or switch to day view
3. View single day's shifts

**Expected Results:**
- [ ] Day view shows detailed daily schedule
- [ ] Hour timeline visible (optional)
- [ ] Shifts show with full details
- [ ] Easy to see breaks/gaps in schedule

---

### TC-9.7.4: Expandable Shift Details

**Steps:**
1. View schedule with shifts on mobile
2. Tap on a shift card
3. Observe expanded details

**Expected Results:**
- [ ] Shift expands to show more details
- [ ] Shows: time, location, role, notes
- [ ] Collapse on tap again or tap elsewhere
- [ ] Smooth expand/collapse animation

---

### TC-9.7.5: My Shifts Focused View

**Steps:**
1. Log in as an employee
2. Navigate to "My Shifts" or toggle to personal view
3. Observe displayed shifts

**Expected Results:**
- [ ] Only shows shifts assigned to logged-in user
- [ ] Hides other employees' shifts
- [ ] Clear view of personal schedule
- [ ] Option to view full schedule (if permitted)

---

### TC-9.7.6: Calendar Month Picker

**Steps:**
1. On mobile schedule view
2. Tap on date/week header or calendar icon
3. Select a different month/date

**Expected Results:**
- [ ] Month picker calendar appears
- [ ] Can navigate between months
- [ ] Selecting date jumps to that week
- [ ] Current date highlighted

---

### TC-9.7.7: Smooth Scrolling Performance

**Steps:**
1. Load schedule with many shifts
2. Scroll rapidly through the list
3. Observe scrolling behavior

**Expected Results:**
- [ ] Scrolling is smooth (60fps target)
- [ ] No jank or stuttering
- [ ] Content loads quickly
- [ ] No blank areas during scroll

---

### TC-9.7.8: Schedule PDF Export

**Steps:**
1. On mobile schedule
2. Find export/download option
3. Export schedule to PDF

**Expected Results:**
- [ ] Export option available on mobile
- [ ] PDF generates correctly
- [ ] Can be saved or shared
- [ ] Contains schedule for selected period

---

### TC-9.7.9: Various Screen Sizes

**Steps:**
1. View schedule on small phone (< 375px width)
2. View on standard phone (375-414px)
3. View on large phone/phablet (> 414px)

**Expected Results:**
- [ ] Layout adapts to screen size
- [ ] No horizontal scrolling required
- [ ] Content remains readable
- [ ] No overlapping elements

---

### TC-9.7.10: Date Range Display

**Steps:**
1. View week view
2. Observe date range header

**Expected Results:**
- [ ] Shows clear date range (e.g., "Jan 15 - Jan 21")
- [ ] Today is highlighted if in range
- [ ] Easy to understand current view period

---

### TC-9.7.11: Empty State Handling

**Steps:**
1. Navigate to a week with no shifts
2. Observe display

**Expected Results:**
- [ ] Clear "No shifts scheduled" message
- [ ] Navigation to other weeks still works
- [ ] No error or broken UI

---

### TC-9.7.12: Shift Color Coding

**Steps:**
1. View schedule with shifts of different types
2. Observe visual differentiation

**Expected Results:**
- [ ] Different shift types have distinct colors
- [ ] Colors match role or location settings
- [ ] Legend available if needed
- [ ] Colors are accessible (not color-only)

---

### TC-9.7.13: Pull-to-Refresh Integration

**Steps:**
1. On mobile schedule view
2. Pull down to refresh
3. Observe result

**Expected Results:**
- [ ] Pull-to-refresh works
- [ ] Schedule data reloads
- [ ] Current view position maintained
- [ ] New shifts appear if any
