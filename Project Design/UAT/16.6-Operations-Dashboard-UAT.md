# UAT: Feature 16.6 - Real-Time Operations Dashboard

## Test Environment Setup

**Prerequisites:**
- Admin user account
- Employees with shifts scheduled for today
- Time clock system active
- Some employees clocked in, some not yet
- Email/notification service configured

**Test URL:** `/operations/dashboard`

---

## Test Cases

### TC-16.6.1: Access Operations Dashboard (Admin)

**Steps:**
1. Log in as admin
2. Navigate to "Operations Dashboard" or "Live View"

**Expected Results:**
- [ ] Dashboard loads successfully
- [ ] Real-time data displayed
- [ ] Current date and time shown
- [ ] Location selector available (if multiple)

---

### TC-16.6.2: Employee Cannot Access Dashboard

**Steps:**
1. Log in as an employee
2. Try to navigate to `/operations/dashboard`

**Expected Results:**
- [ ] Access denied (403 error)
- [ ] Operations dashboard not in sidebar
- [ ] No operational data exposed

---

### TC-16.6.3: "Who's Working Now" View

**Steps:**
1. Have employees clocked in
2. Navigate to operations dashboard
3. View current workers section

**Expected Results:**
- [ ] Shows all currently clocked-in employees
- [ ] Displays employee name and role
- [ ] Shows clock-in time
- [ ] Shows current location/department

---

### TC-16.6.4: Employee Details in Live View

**Steps:**
1. View currently working employees
2. Click on an employee for details

**Expected Results:**
- [ ] Detailed view available
- [ ] Shows shift start/end time
- [ ] Shows hours worked so far
- [ ] Shows break status

---

### TC-16.6.5: Expected Arrivals View

**Steps:**
1. Navigate to operations dashboard
2. View "Expected Arrivals" or "Coming Up" section

**Expected Results:**
- [ ] Shows employees expected in next 1-2 hours
- [ ] Shows scheduled start time
- [ ] Shows role and location
- [ ] Sorted by arrival time

---

### TC-16.6.6: Highlight Late Employees

**Steps:**
1. Have an employee scheduled but not clocked in
2. Wait until past their start time
3. View operations dashboard

**Expected Results:**
- [ ] Late employee is highlighted (red/warning)
- [ ] Shows how late they are
- [ ] Easy to identify at a glance
- [ ] Separated from on-time employees

---

### TC-16.6.7: Highlight Missing Employees

**Steps:**
1. Have employee scheduled for shift
2. Employee does not show up
3. View dashboard after shift start

**Expected Results:**
- [ ] Missing employee shown prominently
- [ ] Clear distinction from late (more severe)
- [ ] Shows missed shift details
- [ ] Quick action to contact available

---

### TC-16.6.8: Quick Contact Actions

**Steps:**
1. View late/missing employee on dashboard
2. Click "Contact" or similar action

**Expected Results:**
- [ ] Contact options available
- [ ] Can send SMS/message (if configured)
- [ ] Can initiate call (click-to-call)
- [ ] Email option available

---

### TC-16.6.9: Coverage Status per Location

**Steps:**
1. Have multiple locations with shifts
2. View operations dashboard
3. Check coverage metrics

**Expected Results:**
- [ ] Coverage shown per location
- [ ] Shows scheduled vs actual staff

- [ ] Easy to identify gaps

### TC-16.6.10: Auto-Refresh Functionality

**Steps:**
1. Navigate to operations dashboard
2. Observe for 2-3 minutes

**Expected Results:**
- [ ] Dashboard updates automatically
- [ ] Refresh happens every minute (or configurable)

- [ ] Updates are visible
