# UAT: Feature 9.9 - Mobile-Specific CSS & Performance

## Test Environment Setup

**Prerequisites:**
- Various mobile devices (low-end to high-end)
- Desktop browser with mobile emulation
- Lighthouse audit tool
- Performance profiling tools

**Test URL:** `/` (any page)

---

## Test Cases

### TC-9.9.1: Safe Area Insets on Notched Devices

**Steps:**
1. Open app on iPhone X or later (or device with notch)
2. View header and footer areas
3. Check both portrait and landscape

**Expected Results:**
- [ ] Content not obscured by notch
- [ ] Bottom navigation respects home indicator
- [ ] Safe area padding applied correctly
- [ ] Works in both orientations

---

### TC-9.9.2: Touch Target Sizing

**Steps:**
1. Using browser dev tools, measure interactive elements
2. Check buttons, links, form inputs

**Expected Results:**
- [ ] All interactive elements minimum 44x44px
- [ ] Adequate spacing between adjacent targets
- [ ] Form inputs are easy to tap
- [ ] No accidental mis-taps

---

### TC-9.9.3: iOS Momentum Scrolling

**Steps:**
1. Open on iOS device
2. Scroll through a long list
3. Perform "flick" scroll gesture

**Expected Results:**
- [ ] Momentum scrolling works smoothly
- [ ] Content continues scrolling after finger lift
- [ ] Natural deceleration behavior
- [ ] No scroll stuttering

---

### TC-9.9.4: Bottom Sheet Animation

**Steps:**
1. Open a modal or bottom sheet on mobile
2. Observe animation
3. Swipe down to dismiss (if supported)

**Expected Results:**
- [ ] Bottom sheet slides up smoothly
- [ ] Animation is 60fps (no jank)
- [ ] Dismiss animation is smooth
- [ ] Background dims appropriately

---

### TC-9.9.5: Image Lazy Loading

**Steps:**
1. Load a page with multiple images
2. Scroll slowly through content
3. Observe image loading behavior

**Expected Results:**
- [ ] Images below fold load on scroll
- [ ] Placeholder shown before image loads
- [ ] No layout shift when images load
- [ ] Improved initial page load time

---

### TC-9.9.6: JavaScript Bundle Size

**Steps:**
1. Open browser dev tools Network tab
2. Load the application
3. Filter for JS files
4. Measure total JS size

**Expected Results:**
- [ ] JS bundle is reasonably sized
- [ ] Code splitting implemented
- [ ] Unused code not loaded
- [ ] Initial load under performance budget

---

### TC-9.9.7: Viewport Meta Tag

**Steps:**
1. View page source or inspect head
2. Check viewport meta tag

**Expected Results:**
- [ ] Viewport meta tag present
- [ ] width=device-width set
- [ ] initial-scale=1 set
- [ ] No maximum-scale restrictions (accessibility)

---

### TC-9.9.8: Performance on Low-End Device

**Steps:**
1. Use a low-end Android device or throttled emulation
2. Navigate through the application
3. Observe performance

**Expected Results:**
- [ ] App loads within acceptable time
- [ ] Animations don't drop frames excessively
- [ ] Interactions are responsive
- [ ] No crashes or freezes

---

### TC-9.9.9: Lighthouse Mobile Audit

**Steps:**
1. Run Lighthouse audit on mobile viewport
2. Review Performance score
3. Review individual metrics

**Expected Results:**
- [ ] Performance score > 70 (target > 90)
- [ ] First Contentful Paint < 2s
- [ ] Largest Contentful Paint < 3s
- [ ] Cumulative Layout Shift < 0.1
- [ ] Time to Interactive < 5s

---

### TC-9.9.10: Touch Manipulation Utilities

**Steps:**
1. On scrollable elements
2. Check if touch-action CSS is applied
3. Verify scroll behavior

**Expected Results:**
- [ ] touch-action: manipulation on interactive elements
- [ ] Removes 300ms tap delay
- [ ] Scroll elements have proper touch-action
- [ ] No conflicts with gestures

---

### TC-9.9.11: CSS Loading Performance

**Steps:**
1. Check critical CSS loading
2. Observe page render

**Expected Results:**
- [ ] No flash of unstyled content (FOUC)
- [ ] Critical CSS inlined or preloaded
- [ ] Non-critical CSS deferred
- [ ] Fast first paint

---

### TC-9.9.12: Font Loading Optimization

**Steps:**
1. Load page and observe text rendering
2. Check for font swap behavior

**Expected Results:**
- [ ] Text visible during font load (font-display: swap)
- [ ] No invisible text during load
- [ ] Minimal layout shift on font load
- [ ] Fonts preloaded if critical

---

### TC-9.9.13: Memory Usage

**Steps:**
1. Open app on mobile device
2. Navigate through many pages
3. Monitor memory usage (if tools available)

**Expected Results:**
- [ ] Memory usage stays reasonable
- [ ] No memory leaks on navigation
- [ ] App doesn't crash from memory pressure
- [ ] Page transitions don't accumulate memory

---

### TC-9.9.14: Orientation Change Handling

**Steps:**
1. Open app in portrait mode
2. Rotate to landscape
3. Rotate back to portrait

**Expected Results:**
- [ ] Layout adapts to orientation
- [ ] No layout breaking
- [ ] Scroll position reasonably preserved
- [ ] No content loss on rotation
